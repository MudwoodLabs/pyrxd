## Executive summary

The strongest signal across personas is the **on-chain RSWP orderbook**: pyrxd already decodes the wire format but has no write/post/fill/browse path, leaving it a spectator on the live book — six personas independently asked for this. The second-largest cluster is **audit-readiness** (mutation testing, residual→test traceability, a scheduled fuzz lane, a real two-party adversarial run) — these are the maintainer's own named gates before non-dust value. Rounding out the high-value work: a **developer-experience facade** (a one-call GlyphClient, typed amounts/errors, importable test fixtures), **watchtower operationalization** (ship it as console scripts + systemd + container, an auto-escalation consumer, a health probe), **dMint mining QoL** (ETA estimator, follow-the-chain head discovery, race-aware re-mine), and a **standards/conformance-vectors program** that turns pyrxd into the reference implementation others differential-test against.

## Top 10 by value-to-effort

1. **Cross-impl HD-derivation differential + published BIP44 vectors** (S/high) — guards the highest-impact silent fund-loss (self-consistent derivation bug across coin-types 0/236/512) that all internal goldens would pass; small, and ships an ecosystem interop asset.
2. **dMint hashrate benchmark + time-to-mint estimator** (M/high, new) — replaces a hardcoded "~1-2M h/s" guess with a real go/no-go signal; pure QoL win every dMint miner hits immediately.
3. **RSWP encoder/builder + post/fill CLI** (M/high) — the most-duplicated request (6 personas); turns pyrxd from orderbook spectator to participant, and it's a serializer+CLI job reusing existing SINGLE\|ANYONECANPAY signing, not new crypto.
4. **Counter-leg-aware `swap status` + read-only `recover-preimage`** (M/high) — closes a one-sided-loss exposure for any operator not running the coordinator; read-only, so it sidesteps the audit gate, and `scrape_secret` already exists on both legs.
5. **Mutation-testing gate on crypto/covenant core** (L/high, new) — the maintainer's named #1 audit-readiness gap; directly answers "would the suite catch a flipped covenant param / swapped sighash flag / off-by-one merkle walk".
6. **Ship watchtower as console scripts + systemd + container** (M/high) — today a `pip install` operator literally cannot run the flagship safety component; encoding the runbook's footguns as shipped defaults converts the most dangerous misconfigs from prose into artifacts.
7. **Auto second-channel escalation monitor** (M/high, partial) — the README's own explicitly-named "open piece"; the producer side (`unacked_critical`) is done, and it covers the worst alert-only failure (a CRITICAL the operator never acks).
8. **Troubleshooting & error index how-to** (M/high, new) — no glossary/troubleshoot/faq page exists; the building blocks (rejection strings, coin_type case, the CLI message/cause/fix block) are scattered, making consolidation the single highest-leverage newcomer artifact.
9. **High-level async `GlyphClient`** (L/high, partial) — every glyph example re-implements ~150 lines of transport + two-pass-fee + reveal boilerplate; the biggest time-to-first-success gap, and the builders already exist underneath.
10. **RSWP conformance vectors + fuzz + golden vector** (M/high, new) — `swap_order.py` is the newest attacker-controlled parser (varint count to 10,000, unchecked CScriptNum) yet the one swap-stack parser with no Hypothesis target, atheris harness, or golden vector.

## Already built / partially built — do not re-do

These items have working substrate in-tree; the work is the missing surface, not the mechanism. Scope to the gap only.

- **RSWP decode side** — `gravity/swap_order.py` fully decodes the v2 frame (and documents the upstream RXinDexer price_terms mis-decode); the wire-format reference doc exists but is untracked. Missing: encoder, orderbook fetch, filler for on-chain orders, vectors, example/how-to.
- **Swap covenant builders & legs** — HTLC covenant/spend builders, `scrape_secret` on both legs, the reorg-finality gate, and single-process e2e regtest all exist. Missing: counter-leg reads, an operator-facing build-and-print CLI, fee escalation, the two-party adversarial harness (design doc drafted), and a versioned recovery-file schema.
- **Partial same-chain swap** — `swap/partial.py` does taker-completion for the private JSON envelope; `AssetKind` is `rxd`/`ft` only with NFTs declared out of scope, and the swap CLI is `status`-only. Missing: NFT support and the offer/accept CLI.
- **Watchtower** — runner scripts, `DedupAlerter.unacked_critical_count`, `FileHeartbeat` (with all the gauges), `presign_refund`, preflight, and the operator/DR runbooks exist. Missing: console-script/systemd/container packaging, the escalation consumer, active endpoint-independence checks, exit-code probe + `schema_version`, metrics exposition, backup/verify commands, and measured tick duration.
- **dMint mining** — parallel miner, deploy/claim commands, `max_height/height` fields, and reroll-on-exhaustion exist. Missing: ETA estimator, follow-forward head discovery (deferred in-code), race-loss re-mine, and initial-difficulty calibration.
- **DX facade** — `RxdWallet`, `GlyphBuilder`, `GlyphScanner`, `RegtestNode`, the `Photons`/`Satoshis` types, `InsufficientConfirmationsError`, the private confirmation-poll loop, and `format_photons` all exist privately. Missing: the networked `GlyphClient`, public `wait_for_confirmation`, typed `InsufficientFundsError`, a public `amount` module, token-balance aggregation, importable `pyrxd.testing` fixtures, and the consolidated `classify_script`.
- **Conformance & standards** — only `conformance/dmint-v2-contract-vectors.json` ships; internal Glyph goldens, the BIP143-quirks how-to, and a token-scoped terminology doc exist. Missing: BIP44/Glyph-envelope/SPV-proof/RSWP/HTLC-SPK vector suites, the normative Glyph spec, the handshake spec, the descriptor export, and the A-Z glossary.
- **Token issuance** — royalty metadata type (decoded but never paid), single-recipient FT transfer, and one-at-a-time NFT mint with the CONTAINER enum present-but-unused. Missing: royalty-honoring transfer, multi-recipient airdrop, and CONTAINER-based collection mint.
- **Governance/network** — `docs/versioning-and-deprecation-policy.md` and a commented-out `FUNDING.yml` exist; a single hardcoded ElectrumX endpoint with connect-time-only racing exists. Missing: the stable/experimental SemVer split + shims, Sponsors + audit fund, the server registry + `for_network()` + request-level failover, and optional TLS SPKI pinning.
